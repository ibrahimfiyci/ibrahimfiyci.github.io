Technical Project

Enterprise LLM Security Gateway & Prompt Firewall Proxy

A FastAPI-based inline proxy for inspecting LLM requests and responses, reducing prompt-injection risk, and enforcing data-loss prevention controls.

Project Overview

This project implements a security gateway between applications and LLM providers. The proxy inspects requests and responses, applies prompt firewall logic, checks semantic similarity against known attack patterns, and redacts sensitive output before it reaches users or downstream systems.

Technologies

  • Python
  • FastAPI
  • Microsoft Presidio
  • FAISS vector database
  • Docker
  • Pydantic

Prompt firewall design

The proxy is designed to sit inline with AI application traffic. It can evaluate incoming prompts before they reach a model and inspect generated responses before they leave the AI layer. This pattern gives development teams a central place to apply security policy instead of scattering checks across many applications.

OWASP LLM risk coverage

The gateway focuses on practical controls for OWASP LLM01 prompt injection and OWASP LLM06 sensitive information disclosure. It uses semantic similarity checks to identify prompt-injection patterns and Microsoft Presidio to detect and redact PII, API keys, and sensitive source-code-like content in outbound responses.

Enterprise integration

The architecture is containerized with Docker so it can be deployed near existing services in an enterprise software development lifecycle. Pydantic models define request and response contracts, while FastAPI provides a low-friction interface for internal application teams.

Why this matters

AI systems often move quickly from prototype to production. A security gateway gives teams a way to monitor AI traffic, apply repeatable controls, and reduce sensitive data exposure while they continue improving the application itself.
