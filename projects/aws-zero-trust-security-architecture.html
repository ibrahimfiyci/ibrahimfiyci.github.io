Technical Project

AWS Zero Trust Security Architecture Lab

A defensive cloud architecture study for least privilege, segmented AWS networks, centralized logging, encryption, and control mapping.

Project Overview

This lab models a secure AWS reference architecture using identity-first access, VPC segmentation, encrypted data services, cloud-native detections, and governance controls that can be explained to both engineering and risk stakeholders.

Technologies

  • AWS IAM, KMS, VPC, WAF
  • CloudTrail, GuardDuty, Security Hub
  • ECS, EKS, Lambda, Fargate
  • RDS, Aurora, DynamoDB, SQS, SNS
  • Terraform-style reference design

Architecture goal

The goal is to show how cloud workloads can be designed around continuous verification: least-privilege IAM, short-lived access, isolated network paths, encrypted storage, and logs that preserve enough context for investigation.

Control design

The design separates identity controls, network controls, data protection controls, workload runtime controls, and monitoring controls. Each layer has a clear reason, an expected log source, and a related security objective.

Study notes

This project connects security architecture with practical cloud engineering. It is useful for studying AWS security, Zero Trust, NIST 800-207 ideas, resilient cloud design, and executive-ready risk communication.
