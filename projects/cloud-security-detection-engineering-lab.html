Technical Project

Cloud Security Detection Engineering Lab

A SIEM-focused lab for building detections from AWS, WAF, endpoint, DNS, and network telemetry while reducing noisy alerts.

Project Overview

This project studies how security teams turn raw cloud and endpoint logs into useful detections. It emphasizes data source quality, ATT&CK-style behavior mapping, alert fidelity, triage notes, and measurable false-positive reduction.

Technologies

  • Splunk Enterprise Security
  • ELK Stack and Azure Sentinel concepts
  • AWS CloudTrail and VPC Flow Logs
  • WAF, EDR, DNS, and endpoint telemetry
  • MITRE ATT&CK mapping

Detection workflow

Each detection starts with a behavior, required telemetry, query logic, expected false positives, enrichment fields, and analyst action. The lab favors clear triage paths over noisy rule volume.

Cloud log coverage

The lab prioritizes CloudTrail, VPC Flow Logs, WAF logs, GuardDuty findings, identity events, and endpoint alerts because cloud attacks often cross identity, network, and workload boundaries.

Study notes

This project supports security engineering, SOC mentoring, alert triage, incident response, and practical security analytics.
