Technical Project

Automated LLM Red-Teaming & Adversarial Jailbreak Framework

A Python-based black-box testing framework for structured evaluation of LLM safety controls, adversarial prompt behavior, and agentic AI risk.

Project Overview

This framework automates controlled red-team test runs against target language models and AI agents. It organizes adversarial test cases, executes repeatable evaluation flows, scores model responses, and produces markdown-based risk assessment reports for review.

Technologies

  • Python
  • LangChain
  • Claude 3.5 Sonnet API
  • Ollama and Llama 3
  • Markdown reporting
  • Git

What the framework tests

The project was designed to evaluate whether a model or autonomous AI agent can maintain intended safety behavior when exposed to adversarial inputs. The validation modules map to AI security themes including agentic goal hijacking, indirect prompt injection, unsafe instruction following, and unauthorized tool-use behavior.

How the system is structured

The framework separates test orchestration, target model calls, evaluation logic, and report generation. This makes it easier to add new test categories, compare model behavior across local and hosted models, and review results without manually copying prompts and responses between tools.

Scoring and reporting

An automated scoring engine parses model outputs for bypass signals, refusal quality, unsafe compliance, and evidence of instruction hierarchy failure. Each run can generate a markdown report that summarizes risk level, observed behavior, and recommended follow-up testing.

Public safety note: This page describes the project at a defensive engineering level and does not publish working jailbreak payloads or step-by-step bypass instructions.
