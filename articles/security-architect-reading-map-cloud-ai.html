Security Architect Reading Map for Cloud and AI Security

By James Fiyci | Updated July 13, 2026

Security architecture sits between technical implementation and risk decision-making. My study map focuses on the areas that connect cloud platforms, identity, telemetry, AI systems, and governance.

Security architecture

A strong architecture study path starts with threat modeling, identity boundaries, segmentation, encryption, monitoring, evidence, and resilience. The question is not only whether a control exists, but whether it can be explained, operated, and verified.

Cloud security engineering

For AWS security, the core topics are IAM, KMS, VPC design, CloudTrail, GuardDuty, Security Hub, WAF, workload isolation, backups, and service-specific risks in containers, serverless, and managed databases.

Detection and response

Detection engineering studies how logs become alerts, how alerts become investigations, and how investigations become better controls. Useful practice includes writing detections, documenting false positives, and connecting behavior to response playbooks.

AI security

AI security adds new concerns: prompt injection, RAG data exposure, model supply chain risk, unsafe tool use, evaluation quality, and governance for autonomous systems. The same architecture habits still matter: least privilege, logging, isolation, and review.

Governance and risk

Security work becomes durable when controls are mapped to risk, evidence, ownership, and review cadence. That is where NIST, MITRE, FedRAMP, SOC 2, ISO 27001, and business continuity discussions become useful.
