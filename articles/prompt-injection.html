AI Security Notes

Understanding Prompt Injection

By James Fiyci | Updated July 10, 2026

Prompt injection is a security problem where untrusted text causes an AI system to follow instructions that the developer or user did not intend. It matters because language models treat many kinds of text as instructions, even when that text came from a webpage, document, email, support ticket, or attacker-controlled input.

Direct and indirect injection

Direct prompt injection happens when a user enters malicious instructions into the AI system. Indirect prompt injection happens when the AI reads outside content that contains hidden or hostile instructions. Indirect injection is especially important for AI assistants that browse websites, summarize files, or use retrieval augmented generation.

Why it is hard

Traditional software can separate code from data more strictly. LLM applications often mix developer instructions, user requests, retrieved documents, and tool results inside one context. The model then has to decide what to trust, which is not the same as a security boundary.

Safer design patterns

  • Do not give the model broad access to sensitive data or powerful tools.
  • Use allowlists for tool actions and require confirmation for risky operations.
  • Label untrusted content clearly and keep it separate from system instructions.
  • Validate outputs before using them in code, commands, emails, or API calls.
  • Log model decisions and test with adversarial examples.

Bottom line

Prompt injection is not solved by one clever prompt. It requires secure application design: least privilege, data boundaries, verification, monitoring, and careful control over what actions an AI system can take.
