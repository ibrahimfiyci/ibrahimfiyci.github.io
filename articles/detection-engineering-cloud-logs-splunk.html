Detection Engineering with Cloud Logs and Splunk

By James Fiyci | Updated July 13, 2026

Good detection engineering is a loop: understand behavior, confirm telemetry, write logic, test false positives, document triage, and improve the control after incidents and reviews.

Start with behavior

A useful detection begins with a security behavior, not a tool feature. Examples include suspicious identity changes, unusual API activity, abnormal network paths, endpoint execution, or WAF events tied to application risk.

Know the data sources

AWS CloudTrail, VPC Flow Logs, WAF logs, GuardDuty findings, EDR alerts, DNS telemetry, and authentication logs each answer different questions. Detections become stronger when they combine sources without creating unreadable logic.

Make triage clear

Every alert should explain why it fired, what evidence matters, what false positives are expected, and what an analyst should check next. That is how SIEM content becomes usable security work.
